Жадібний алгоритм працює швидше,
оскільки проходить по списку монет лише один раз.

Алгоритм динамічного програмування
гарантує оптимальний результат для будь-якої системи монет,
але потребує більше пам'яті та часу.

Для набору монет [50,25,10,5,2,1]
обидва алгоритми дають однаковий результат,
але жадібний працює швидше.
